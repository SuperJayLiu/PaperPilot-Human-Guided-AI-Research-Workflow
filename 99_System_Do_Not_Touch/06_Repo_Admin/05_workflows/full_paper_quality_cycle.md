# Full Paper Quality Cycle

This internal workflow supports the user-facing `01_Start_Here/Run_Full_Paper_Process.md`.

## Core rule

A paper is not complete when it has a complete draft. It is complete only when paper-level contradictions, unsupported claims, data/method mismatches, and disclosure gaps have been found, fixed, or explicitly classified.

## Three review cycles

1. **Consistency and errors**: check whether all parts of the paper say the same thing and whether the evidence supports each claim.
2. **Contribution and reviewer risk**: check novelty, field positioning, identification/design threats, missing central evidence, and likely referee objections.
3. **Final readiness**: check writing polish, references, tables, figures, logs, data access records, AI-use records, and residual issue classification.

## Do not ask twice

Before every cycle, read memory and records. Reuse stable facts. If facts conflict, ask for confirmation and log the correction.

## Whole-paper novelty check

Each whole-paper review cycle must check whether the final paper preserves the approved contribution. The final draft should neither hide the novel angle nor overclaim it. Use `03_Record_Required_Evidence/10_Novelty_And_Mainstream_Deviations/` as the audit trail.
