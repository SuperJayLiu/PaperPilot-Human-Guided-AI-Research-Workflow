# Resource Influences

PaperPilot uses the same design principle as the AI-for-Economics-and-Finance-Research resource section:

```text
source -> workflow pattern -> economics/finance adaptation -> copy-ready asset -> verification rule
```

## Patterns used here

| Source pattern | PaperPilot adaptation |
| --- | --- |
| Keep only primary working folders visible | Top level shows only start files, `02_Build_The_Paper/`, `03_Record_Required_Evidence/`, and `99_System_Do_Not_Touch/` |
| Official tool docs matter | Codex and Claude Code have separate front-door files |
| Data safety map | Data must be classified before AI inspection |
| Skill repositories | Skills have triggers, workflow, failure modes, outputs, and verification checklist |
| Economist agent workflows | Plan Mode precedes Action Mode; humans approve decision packets |
| Writing and review workflows | Claims must map to evidence before drafting or revision |

## Resource categories to check during real projects

- official Codex and Claude Code docs;
- data-provider documentation and license terms;
- canonical method references;
- field-specific finance/economics workflows;
- journal, conference, university, employer, IRB, and coauthor rules.
