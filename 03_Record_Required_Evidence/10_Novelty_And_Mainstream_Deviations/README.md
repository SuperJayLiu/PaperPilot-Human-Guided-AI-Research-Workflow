# Novelty and Mainstream-Deviation Records

Use this folder when the project has a contribution that may deviate from the mainstream literature.

The purpose is **not** to force every paper to be contrarian. The purpose is to prevent the agent from flattening a genuinely novel idea into a safe mainstream summary, while still requiring the idea to be justified, testable, and bounded.

## Core rule

```text
Deviation from the mainstream is not a flaw by itself.
Unsupported deviation is a flaw.
```

The agent must preserve a promising unconventional idea long enough to evaluate it carefully. It may recommend rejecting or narrowing the idea only after writing down:

1. the mainstream view;
2. the proposed deviation;
3. why the deviation is plausible;
4. what evidence would support it;
5. what evidence would falsify it;
6. what a skeptical referee would object to;
7. how the paper can phrase the claim without overclaiming.

## Files in this folder

| File | Purpose |
| --- | --- |
| `novelty_positioning_register.csv` | tracks the paper's novelty claims, mainstream alternatives, and evidence status |
| `mainstream_deviation_register.csv` | tracks where the paper intentionally deviates from standard framing, data, method, or interpretation |
| `novelty_referee_objection_log.csv` | records skeptical objections and planned responses |
| `originality_preservation_checklist.md` | checklist to prevent the agent from over-normalizing the paper |
| `deviation_packet_template.md` | copy-ready packet for deciding whether to keep, narrow, test, or drop a non-mainstream idea |

## When this folder must be updated

Update this folder during:

- research question selection;
- literature positioning;
- empirical design;
- finding interpretation;
- writing and claims;
- reviewer stress test;
- whole-paper review cycles.

If the paper does not deviate from the mainstream, record that explicitly. The goal is transparency, not contrarianism.
