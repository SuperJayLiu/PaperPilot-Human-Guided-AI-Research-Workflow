# Full Paper Quality Cycle Records

This folder records the end-to-end paper review layer.

Use it after the project has enough material for a complete draft, or when the researcher requests a full paper run from idea to final output.

## Required records

| File | Purpose |
| --- | --- |
| `full_paper_run_status.csv` | stage-by-stage status for the full paper run |
| `whole_paper_issue_register.csv` | all issues from whole-paper reviews and their status |
| `review_round_template.md` | copy-ready template for each paper-level review round |

## Three-cycle rule

A complete run should include at least:

1. consistency and error check;
2. contribution and reviewer-risk check;
3. final polish and submission-readiness check.

Each issue must end as fixed, not applicable, documented limitation, accepted human-author judgment, future work, or blocked and logged.

## Novelty-preservation during whole-paper cycles

Each whole-paper review cycle must ask whether approved originality has been erased, under-explained, or overclaimed. Use `../10_Novelty_And_Mainstream_Deviations/` to check the mainstream benchmark, proposed deviation, evidence status, and referee objections.
